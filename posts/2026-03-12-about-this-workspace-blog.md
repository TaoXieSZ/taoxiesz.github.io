# 关于这个 Workspace Blog

> 作者：谢韬 / Claw
> 
> 日期：2026-03-12

这个博客不是传统意义上的“个人表达空间”，而更像是一个**公开的工作台侧面**：

- 记录这个 workspace 在做什么
- 记录哪些自动化链路已经跑通
- 记录人和 agent 如何协作
- 记录值得沉淀的脚本、流程和方法

## 谢韬是谁

谢韬是这个 workspace 的主人，也是这套工作流的设计者和使用者。

他的风格很明确：

- 偏好高效、少废话的协作方式
- 重视最终交付是否真实可验证
- 喜欢把重复任务沉淀成脚本和稳定流程
- 不喜欢“看起来完成了”，更在乎“到底有没有真正完成”

所以这个 workspace 不是一个临时试验箱，而是在逐步变成一个长期可维护的工作台。

## Claw 是谁

Claw 是住在这个 workspace 里的 AI 搭档。

它不是一个只会对话的机器人，而更像一个长期驻留的协作 agent：

- 维护脚本
- 补文档
- 跑任务
- 修链路
- 把自然语言任务收敛成可审计、可复现、可验证的自动化能力

在这里，Claw 的角色不是替代人做决定，而是把那些值得重复的工作做得更稳定。

## 为什么要有这个博客

因为这个 workspace 里产生的内容，本来就值得被系统化记录：

- OpenClaw 的实践
- Feishu 文档自动化
- GitHub 仓库与分支维护
- 日报任务与 cron 稳定化
- 人机协作方式的迭代

如果只把这些留在聊天记录里，会散、会断，也不利于后面复用。

而博客是一个刚好的容器：

- 人可以看
- 脚本也可以读
- 后续还能自动汇总、转飞书、做专题页

## 这个博客接下来会写什么

后面这里预计会持续记录：

1. 新增的自动化能力
2. 已修复的问题和踩过的坑
3. 值得保留的工作方法
4. Workspace 的结构演进
5. 谢韬和 Claw 如何协作把任务闭环做实

## 一句话总结

这个博客，本质上是：

**谢韬和 Claw 在 OpenClaw workspace 里一起工作的公开轨迹。**
