今天把个人博客的方向正式收敛了：

- 框架选 **Astro**
- 托管选 **GitHub Pages**
- 内容格式用 **Markdown**
- 目标是“长期可维护的公开工作台”

## 为什么是这个组合

因为它同时满足几件事：

- 足够轻
- 足够稳
- 容易长期维护
- 部署链路清楚，可验证

## 第一版先不追求什么

- 不追求复杂交互
- 不追求花哨视觉
- 不追求 CMS

先把这些打稳：

1. 首页
2. 博客列表
3. 文章页
4. GitHub Actions 自动部署

## 接下来要补的

后面可以继续加：

- 自定义域名
- RSS
- SEO 元数据
- 项目页细化
- 评论或反馈入口

但第一优先级还是：**先持续写起来。**
