## 为什么是 Cursor

为什么有这个标题呢，主要就是因为工作单位当前只允许我们使用 Cursor 干活。幸运的是，我司购买的商业版支持我们使用各种模型，包括 Opus 4.6等等，而且额度也很足够，只要不开 max 随便花。当然，我个人也是有用到 CC 和 codex，在使用上都有共同之处。

## Cursor vs Claude Code

claude code 当然强大好用，但是涉及到前端页面，跟它的交流就会变得非常费劲，打个比喻，当你要给你的前端同事提需求，但是他不在你身边，想象一下：

- 你只能用微信给他发截图
- 得等他改完发版本才能看到改成什么样
- 你得在浏览器和微信互相切换

……

怎么样，是不是血压一下就上来了。当然，我也探索过使用cmux,把终端和浏览器放一起，但是发截图交流还是个困难的事情。

但是写界面还是 Cursor 比较爽。因为 Cursor 自带浏览器，同时可以用`select-element` 直接把要改的元素发到agent 对话框里，不需要截图。当然浏览器支持截图，点击后直接贴在对话框。反正怎么样都能比 cc 方便。
