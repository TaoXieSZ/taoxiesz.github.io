这个博客不是传统意义上的“个人表达空间”，更像一个**公开工作台**。

它主要用来记录三类东西：

- 技术与自动化实践
- 项目与工作流复盘
- 值得长期沉淀的方法

## 为什么要有它

因为很多真正有价值的内容，如果只停留在聊天里，会散、会断，也不方便复用。

博客是一个刚好的容器：

- 人可以读
- 以后脚本也可以处理
- 可以持续沉淀成自己的公开档案

## 这个博客会写什么

后面最值得持续写的方向包括：

1. OpenClaw 与 agent 协作实践
2. Feishu 文档自动化
3. GitHub 仓库维护与发布流程
4. 自动化链路里的踩坑和修复
5. 个人工作流的工程化整理

一句话说，这里会更像**公开工作台的记录面**，而不是花哨展示页。
