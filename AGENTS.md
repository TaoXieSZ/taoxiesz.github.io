# AGENTS.md

This file provides guidance to WARP (warp.dev) when working with code in this repository.

## Project overview
- This is an Astro 5 static site deployed to GitHub Pages.
- The active app lives under `src/` and is built with `astro build`.
- Blog content is sourced from Astro Content Collections (`src/content/blog/**`), not from the legacy `posts/` directory.

## Commands
- Install dependencies: `npm install`
- Start local dev server: `npm run dev` (alias: `npm run start`)
- Build production site: `npm run build`
  - This runs Astro build, then indexes output with Pagefind (`npx pagefind --site dist`) for `/search`.
- Preview built output: `npm run preview`
- Validate project types/content/frontmatter: `npm run check` (`astro check`)

## Test and lint reality in this repo
- There is currently no dedicated unit test runner (no `test` script) and no standalone linter script.
- `npm run check` is the primary validation command for schema/types/content correctness.
- “Single test” execution is not applicable until a test framework is added.

## Architecture (big picture)
### Content and schema
- `src/content.config.ts` defines one `blog` collection with a Zod schema (title/description/dates/tags/author/project/draft).
- `draft: true` entries are excluded everywhere via `getCollection('blog', ({ data }) => !data.draft)`.
- `project` is constrained to known values and drives project-specific routing/filtering.

### Rendering pipeline
- Listing pages (`src/pages/index.astro`, `src/pages/blog/index.astro`, `src/pages/projects.astro`, `src/pages/tags/index.astro`) read from the same collection and derive views by sorting/grouping/filtering.
- Post detail pages are generated by `src/pages/blog/[...slug].astro` using `getStaticPaths()` + `render(post)`.
- Post URLs depend on `postSlug()` in `src/consts.ts`, which strips directory and `.md` extension from content IDs.
- RSS feed (`src/pages/rss.xml.ts`) is generated from the same blog collection, keeping feed and site content aligned.

### Layout/system boundaries
- `src/layouts/BaseLayout.astro` centralizes SEO metadata, canonical URLs, OpenGraph/Twitter tags, nav, theme toggle, and global styles.
- `src/consts.ts` is the single source for site metadata (`SITE_TITLE`, `SITE_DESCRIPTION`, `SITE_URL`), nav links, and project taxonomy (`PROJECTS`).
- Shared UI building blocks are in `src/components/` (`PostCard`, `TableOfContents`, mascot).
- Reading time logic is centralized in `src/utils/reading-time.ts` (mixed CJK/Latin estimation).

### Search and deployment coupling
- `/search` (`src/pages/search.astro`) depends on static assets generated by Pagefind during `npm run build`.
- GitHub Actions deployment is defined in `.github/workflows/deploy.yml` and runs on pushes to `main`:
  - Node `22.14.0`
  - `npm install --no-audit --no-fund`
  - `npm run build`
  - deploys `dist/` to GitHub Pages

## Important repo-specific caveats
- Root-level `index.html`, `manifest.json`, and `posts/*.md` appear to be a legacy static blog format and are not part of the Astro content pipeline.
- If you are changing live blog behavior, prioritize files under `src/` and `src/content/blog/`.
- `src/pages/tags/[tag].astro` is currently empty; tag index links exist, but tag detail behavior is not implemented in this file yet.
